## The four decisive findings

1. **Identity lives in supported editor state — no plugin.** Rung (a)
   (built-in UUIDs/labels) is stable but cannot carry role/binding/slot/
   variant; rung (b) — a `Semantic` user component — holds all five fields,
   survives every editor operation, and compiles into runtime code as plain
   assignments. The ladder stopped there.
2. **Headless regeneration is unsupported** (recorded limitation, not a
   blocker). The scene compiler exists only in the workbench browser client;
   the server binary and MCP bridge expose no compile command. Compensations:
   the editor auto-compiles on save (scene and generated output move
   together), generation is byte-deterministic, and the workbench itself is
   scriptable against the local server (that is GUI automation, recorded as a
   vendor cost for goal-U7's parity adapter — not headless support).
3. **Safety posture holds.** The compiler emits hostile strings as correctly
   escaped double-quoted literals (including `<\/script>` XSS-safe encoding);
   AST tests prove hostile content never escapes a string literal into code,
   comments, or template expressions.
4. **Worktree confinement is clean (KTD11).** Across all recorded sessions
   the editor wrote nothing outside the fixture; the dotted `.twf-worktrees`
   path was accepted. No duplicate-checkout environment is needed.

## What goal-U1 should take

- Pins: `phaser@4.2.1` exact; toolchain pins with resolved versions in
  `report.json` `dependencies`.
- Editor artifacts commit as: `.scene`/`.components`/asset-pack/config =
  authority; generated `.ts` = hash-pinned derived output (regenerable only
  via editor GUI — see finding 2).
- R14a command-surface: all six shared lane commands (`validate`, `publish`,
  `preflight`, `apply`, `status`, `proof`) are scriptably backable and map
  onto the shared typed vocabulary; drift detection must pin committed
  generated bytes by hash rather than re-deriving them headlessly.
- The only justified plugin is the 26-line live-preview forwarder; identity
  and catalog UX need none.

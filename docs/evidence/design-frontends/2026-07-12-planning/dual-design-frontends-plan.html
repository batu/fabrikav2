A controlled comparison of GrapesJS + DOM/CSS and Phaser Editor + Phaser-native rendering for the same seven-surface mobile game shell. This page is a review view; goal.md is the canonical plan.

One visual authority per proof game

GrapesJS project state owns the Grapes proof. Phaser Editor scene state owns the Phaser proof. The shared contract constrains semantics, bindings, assets, and geometry; it does not carry a chosen visual design.

No decision is baked in

The existing Grapes work is preserved, but neutral P0 and scored starts come from the same frozen seed. Build cost and post-completion usability are reported separately.

Two complete lanes

Same shell. Different authority-to-runtime seam.

Both lanes implement Menu, Level, Shop, Settings, Pause, Win, and Fail, including fake commerce, a second currency, curated assets, live copy, direct manipulation, and stable duplication.

GrapesJS → DOM/CSS

  • Finish the constrained editor already in flight.

  • Prove stable identity, live copy, safe code generation, and offline exit.

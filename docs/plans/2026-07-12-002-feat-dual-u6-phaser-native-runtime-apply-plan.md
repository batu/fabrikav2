# Phaser-native Shell Runtime + Immutable Apply (DUAL U6) - Plan

## Goal Capsule

- **Objective:** Render the functional seven-surface shell (`menu, level, shop, settings, pause, win, fail`) entirely through Phaser 4, bound to the same frozen `TemplateShellController` + fake `TemplateSdk` the DOM lane uses, and prove the identical immutable **P0 / A / B / B** application contract — the contract-default P0 baseline is reproduced, a valid A applies, a distinct B over A applies, and reapplying B is a true byte/mtime/dir-entry/pointer/git-status no-op — using the same typed outcome vocabulary as the Grapes/DOM lane. U6 consumes U5's three accepted immutable publications (P0, A, B) as its exact inputs and never fabricates authoring fixtures.
- **Product authority:** Batu approves the experience side-by-side on the physical Android device (a later U7/U8/U10 gate). This card owns the Phaser **runtime / projection / application** lane only; it consumes U5's accepted `phaser-native` publications and never re-authors them. The landed `shell-presentation-v2` kernel contract owns geometry, identity, publication/projection IDs, normalization/projection/caps, and profile artifact rules; U6 reuses them and introduces no third design document. Accepted publication presentation — not U6 — drives geometry, copy, assets, colors, order, visibility, variants, and duplicates; the contract `defaultPresentation` constructs and verifies **P0 only**.
- **Execution profile:** Deep, single-lane implementation with two hard upstream dependencies asserted at preflight (U1 final seal/fence/de-cycle `qWCv9tUo`; U5 accepted P0/A/B publications `gJtZP63y`). All work lands to `experiment/dual-design-frontends`. The conductor runs the first live Phaser-in-WebGL shakedown; the U6 local shakedown is **real-browser explicit-WEBGL only** — all Android/tunnel/reload-timing/tap work stays in U10.
- **Stop conditions:** Stop with a named `comparison_blocked` / `no-go` handoff — never a workaround — if: the U1 final seal/fence/de-cycle prerequisite has not landed; U5 has not handed off the three accepted P0/A/B publicationIds; Phaser 4 explicit-WEBGL cannot boot in the target environment (no `AUTO` fallback); or texture/font readiness cannot be observed deterministically before `ready`. Do not begin U10 device bridging, warm-propagation timing, or scored sessions here.

## Product Contract

### Summary

Turn U5's three accepted immutable `phaser-native` publications into immutable DOM-parity runtime revisions, select exactly one through an atomic pointer, and draw the whole shell in one persistent explicit-WEBGL `Phaser.Game` that boots from that pointer. Revision **promotion is a controlled page reload** (goal KTD14) — no in-place cross-revision module/scene swapping — while a single `Phaser.Game`/canvas persists across state transitions within a page lifetime. The runtime exposes renderer-neutral evidence (state, revision, CSS-client action rectangles, post-paint readiness) without any DOM shell. The behavior source (controller, SDK, catalog, harness, in-situ tour) is a read-only lane input; U6 only re-projects it through Phaser and proves the deterministic apply loop.

### Problem Frame

The Phaser lane must be behaviorally and semantically indistinguishable from the Grapes/DOM lane at the contract level, while being a genuinely different renderer (canvas, not DOM). Three hazards define the work:

1. **No DOM crutch.** The current `games/shell_proof_phaser/src/shell/` is the U1-seeded **DOM** renderer (`TemplateShell.ts` + `template-shell.css`) frozen byte-identical to the Grapes twin. Reusing any of it — or exposing `data-fab-*` DOM hooks as the evidence source — is "DOM-shell reuse disguised as Phaser-native" and is disqualifying. The Phaser renderer lives only in `games/shell_proof_phaser/src/shell/renderers/` (plus the exact U1-opened `main.ts` / `TemplateShell` / CSS entry paths) and derives the same action identities and rectangles from **live Phaser input hit areas**, not DOM.
2. **Immutable apply parity.** U6 must reproduce, for the `phaser-native` profile, the same fail-closed loop the DOM lane (U4) implements — verified publication ingest → non-executing load gate → immutable `design/revisions/<projectionId>/` → atomic `design/revision.json` → deterministic ledger → drift audit — with a byte/mtime/dir-entry/pointer/git-status no-op on B-over-B and the negative asset-identity control preserving the prior revision. The canonical projected artifact is exactly `scenes/shell.js` plus a closed `scene-manifest.json` mapping the seven state exports, governed by a defined **RevisionSource ABI** that yields identical content-addressed bytes in dev and dist.
3. **Deterministic readiness on an async renderer.** Phaser loads textures/fonts asynchronously and re-renders on its own clock. Every render transaction is keyed to `(projectionId, state, renderGeneration)` — including same-state settings toggles, currency changes, and Shop callbacks — and `ready` is asserted only after zero loader failures, all required resources/actions/bindings resolved, Shop init settled, a visible sentinel, and the next `POST_RENDER` for that generation. Bounded terminal-failure behavior and recovery are defined so a caller never hangs on `ready=false` forever.

## Planning Contract

### Key Technical Decisions

- **KTD1 — One persistent explicit-WEBGL `Phaser.Game`/canvas per page lifetime; promotion is a controlled page reload.** Boot a single `Phaser.Game` with `type: Phaser.WEBGL` (no `AUTO`) and the Scale Manager in `FIT` portrait against the canonical 390×844. The single game/canvas persists across all seven **state transitions** within a page lifetime; it is never destroyed/recreated on presentation refresh. **Revision promotion (A→B) triggers a controlled page reload** (align with goal KTD14: reload without target build, sync, install, or relaunch), which re-boots the one game against the newly promoted pointer. **Remove** in-place cross-revision module/scene swapping and per-`sourcePublicationId` runtime key namespaces from rev1 — a page load resolves exactly one revision.
- **KTD2 — Pull-model renderer bound through the frozen `harness.render` callback.** The controller is pull-only (`snapshot()` + boolean verbs, no observer). The Phaser renderer exposes a `render()` that re-reads `snapshot()` and repaints the active scene; it is passed to the frozen `createTemplateHarness({ render })` and invoked after every mutating verb — exactly the DOM lane's `controller.<verb>(); render();` discipline, with zero controller/harness/insituTour byte changes.
- **KTD3 — One geometry pipeline: project once, FIT once, probe on the same matrix.** Authored canonical baseline-safe geometry (from the accepted publication's per-instance `defaultPresentation.geometry`, normalized via `normalizeShellGeometry` where authored px need it) is projected **once** through `projectShellGeometry({ anchor, geometry, viewport, caps, assetSize })` against `canonicalCanvas` (390×844, `baselineInsets`, `baselineSafeRect`), then FIT **once** to the measured CSS safe viewport. The evidence probe reads action rectangles through the **same matrix** — never a parallel geometry path. Per-role `minimumTouchTarget: 48` holds; no double inset.
- **KTD4 — Action rectangles from live Phaser hit areas, in CSS-client space.** Each rectangle derives from the actual Phaser input hit area intersected with the active scene, ancestor containers, alpha, camera, canvas placement, masks, modal/input occlusion, and (for Shop) the catalog viewport, then projected through the KTD3 matrix into **CSS-client pixels** (the coordinates a real pointer event carries). Each emits the required `actionHook` plus a semantic `instanceId`; `devicePixelRatio` appears only in `viewport`. Actions are addressed by `(actionId, instanceId)`; duplicate `play` actionIds across instances are legitimate. Tested against nested/scaled/rotated/masked/offscreen/duplicate actions with no double inset and no world/backing-store/DPR-multiplied coordinates.
- **KTD5 — Readiness keyed to `(projectionId, state, renderGeneration)` with bounded terminal failure.** A `renderGeneration` counter increments on every render transaction, including same-state settings toggles, currency changes, and Shop callbacks. `ready` returns true only after: zero loader failures, all required resources/actions/bindings for the state resolved, Shop `iap` init settled (for `shop`), the visible revision sentinel drawn, and the next `POST_RENDER` for the current generation. Every async texture/font/Shop callback checks its captured generation before mutating readiness or the scene, so a stale result cannot satisfy a newer state. **Terminal failure is bounded and recoverable:** a defined loader/timeout budget flips the transaction to an explicit terminal `failed(reason)` state (surfaced to `status`/probe) instead of leaving callers on `ready=false` forever.
- **KTD6 — RevisionSource ABI; canonical artifact is exactly `scenes/shell.js` + closed `scene-manifest.json`.** Define a `RevisionSource` ABI: the projected canonical artifact is exactly one `scenes/shell.js` module whose closed `scene-manifest.json` maps the seven state exports (`menu, level, shop, settings, pause, win, fail`) to their entry symbols, with `asset-pack.json`, non-circular `asset-identity.json`, and source rasters alongside. The ABI produces **identical content-addressed bytes in dev and dist** so the same revision loads under Vite dev and the bundled build. No user-code/import regions, symlinks, paths/URLs, source↔generated divergence, or volatile bytes.
- **KTD7 — Non-executing load gate before any module import.** Before the runtime imports `scenes/shell.js`, a non-executing gate revalidates: the pointer, the exact artifact set, hashes/byte counts, an AST allowlist (declarative scene data only — no user code/imports), paths, symlinks, URLs, and the immutable content-addressed URL. Only after the gate passes does the runtime import and execute. Production dist must clear the same gate and boot B with the source checkout unavailable.
- **KTD8 — Apply: no-follow copy → private same-filesystem staging → validate staged bytes → derive id → lock + compare → atomic promote.** `apply` copies the accepted publication **no-follow** (never following symlinks) into a private same-filesystem staging dir, validates the **staged bytes only**, derives the `projectionId` there via `computeShellProjectionIdV2` (binding `sourcePublicationId`), takes an exclusive apply lock, does an expected-current compare (guarding pointer/target drift), then **atomically promotes** the immutable `design/revisions/<projectionId>/`, `design/revision.json`, and the ledger together. Tested against source mutation during apply and concurrent A/B applies. Any drift or expected-current mismatch → `blocked-drift`; failure leaves the prior pointer active.
- **KTD9 — B-over-B no-op is measured, not assumed.** The second B compares before/after **bytes, mtimes, directory entries, pointer, and a git-status snapshot** and asserts they are unchanged; it does **not** require an absolute-clean working tree after B1 (only that B2 changes nothing relative to the post-B1 state). No writes, no pointer rewrite.
- **KTD10 — One `tools/phaser-shell` CLI.** Keep a single CLI. It composes U5's `validate / publish / preflight / status / proof` verbs with U6's projection/apply internals and adds **only** the new public `apply` command. U6 does not fork a second CLI or reimplement U5's verbs. Typed outcomes `applied / no-op / blocked-drift / invalid-revision / unsupported-intent`; kernel `ShellValidationIssue` codes map into `invalid-revision` / `unsupported-intent`; drift → `blocked-drift`.
- **KTD11 — Individual source raster identity preserved.** Load approved individual raster bytes per `asset-identity.json` (`{ instanceId, slotId, assetId, path, sha256 }`); do not atlas or derive textures unless U2 proved a deterministic identity-preserving derivative is required (it did not; default to individual bytes). The negative asset-identity control (swap a raster's bytes under a valid-looking binding) must fail the drift audit and preserve the prior revision.
- **KTD12 — Evidence probe is debug/evidence-build only and sanitized.** `createShellEvidenceProbe({ gameId, contractId: "shell-presentation-v2", rendererProfile: "phaser-native", readers })` is registered **only in the debug/evidence build**, frozen non-enumerable/non-configurable/non-writable on `window[evidenceProbeWindowKeyForGame(gameId)]`, and **absent from bundled comparison/release builds**. Every `status`/`proof`/`ledger`/`evidence` output is sanitized (no license/credential/machine-path/volatile bytes).
- **KTD13 — Renderer-local references under `refs/runtime/<projectionId>/`.** Generate `phaser-native` expected references under `games/shell_proof_phaser/refs/runtime/<projectionId>/`, hash-bound to publication id, projection id, renderer fingerprint, viewport, and safe-area profile, before any device run. (U5 owns `refs/authoring/<publicationId>/`.) Consumers reject a reference of the wrong purpose/identity/profile/viewport/safe-area; a judged device capture can never become its own expected reference, and revision A's references cannot verify B.
- **KTD14 — Presentation authority + accessibility preserved.** The accepted publication's presentation drives geometry, copy, assets, colors, order, visibility, variants, and duplicates; the kernel normalizes/projects/caps those authored facts; the contract `defaultPresentation` constructs/verifies **P0 only**. Accessibility metadata and action semantics carried by the publication are preserved and validated end-to-end (into action rectangles/`actionHook`s); U6 adds **no visual DOM shell**, and a full screen-reader overlay stays out of scope unless the origin contract is separately expanded.

### High-Level Technical Design

**Two cooperating pieces:** (1) the **application lane** in `tools/phaser-shell/src/application/` (ingests accepted publications, load-gates them, projects to immutable revisions, and atomically selects one) surfaced through the single `tools/phaser-shell` CLI; and (2) the **runtime renderer** under `games/shell_proof_phaser/src/shell/renderers/` (boots one WEBGL game from the selected pointer and draws the shell). They meet at the on-disk `design/revision.json` pointer + `design/revisions/<projectionId>/` bundle. Per KTD1, the runtime reads the pointer once per page load; promotion is a controlled reload, not a live swap.

Frozen reuse vs. lane-owned (verified against `experiments/design-frontends/baseline/behavior-hashes.json`):

| File / surface | Role | U6 disposition |

| `src/sdk/TemplateSdk.ts`, `src/sdk/proofShopCatalog.ts` | fake SDK + catalog | reuse byte-for-byte |

**Runtime architecture:** a boot module reads `design/revision.json`, runs the non-executing **load gate** (KTD7), then imports the gated `scenes/shell.js` and registers the seven scenes per the closed `scene-manifest.json`; the single WEBGL `Phaser.Game` (KTD1) persists across state transitions; a surface router maps `snapshot.surface` to the active scene and repaints on `render()`; a semantic layer binds each interactive display object to `(actionId, instanceId)` with `disabled/visible` from snapshot state and wires pointer handlers to the controller verbs; a Shop scene consumes `controller.sdk.iap` across its full state matrix (below) with generation-guarded results; a probe adapter (debug build only) implements the evidence readers on the one geometry matrix; a scene-isolation wrapper ensures one scene's failure cannot prevent others or corrupt the evidence bridge, and drives the bounded terminal-failure path (KTD5).

**Shop state matrix (mirrors frozen behavior):** `initializing`, `ready`, `empty`, `unavailable`, and restore sub-states `restore-pending / restored / empty / failed`. Header, Back, and balances are fixed; the catalog alone scrolls/clips; each state has explicit enabled/copy/retry/readiness rules, generation-guarded so a late `iap` callback cannot satisfy a newer state.

**Application-lane architecture (single CLI, KTD10):** `preflight` (U5-composed: resolve the explicit `publicationId`, re-validate the bundle against the `phaser-native` profile via `parseProjectionRevisionV2`/`parseShellPublishedRevisionV2`, check state-family completeness + required actions + accessibility + safe-area after anchor mapping, block before writes); the new public `apply` (KTD8: no-follow stage → validate staged bytes → derive `projectionId` → lock + expected-current compare → atomic promote of revision/pointer/ledger, RevisionSource ABI producing `scenes/shell.js` + closed manifest); `status` (U5-composed: report current pointer/revision/outcome, sanitized); `proof` (U5-composed: regenerate + byte-diff drift audit, negative controls). Typed outcomes `applied/no-op/blocked-drift/invalid-revision/unsupported-intent`.

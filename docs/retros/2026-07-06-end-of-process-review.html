What the numbers say

One page from the evaluation ledger (board.jsonl, 319 records) and twf metrics report (fabrikav2, 22 cards). Every claim is reproducible — see the appendix. Two findings dominate; three caveats bound what the data can support.

aRework concentrated in first-of-kind work; ports & composition needed noneHeadline

Of 29 outcome-bearing land records, 0 cards needed more than one rework round. 21 (72%) needed zero; 8 (28%) needed one. The 8 that reworked are all scaffolding, cross-package generalization, or first-consumer integration — cards defining a contract or seam for the first time. The 21 clean cards are pure ports, composition-from-existing-primitives, additive wiring, and the design-sheets CLI.

Quantitative support for F4 (shared registration points collide) and F5 (cross-package contracts need a declared owner). Fixes #3, #5 target the measured cost.

bCeremony is a ~1-second march through 6–8 columnsHeadline

Real work lives in worked (minutes–hours). Every downstream gate — tested_insitu, reviewed, evidence, video, compounded — is a 1-second twf next + comment on one-pass cards. The mechanical confirmation of F1 ("~30% of conductor actions were column-marching") and F16 (an impl card forced through a brainstorm-doc checklist). This very card hit F16: the raw todo→brainstormed transition demands a requirements doc that makes no sense for an analysis deliverable.

Fixes #2, #7 collapse non-applicable gates by card classification.

cRouting happened; routing evaluation can't be computed yetCaveat

279 spawns carry a routed worker model (gpt-5.5 42%, opus-4-8 37%, fable-5 6%). But all 29 outcome records are attributed to claude/opus (the conductor doing land bookkeeping) — not the model that did each stage. So task-class × worker-model performance, exactly what idea #14 (route-suggest) needs, is not joinable today. The one readable cross-vendor outcome: MQPvX0qi (reskin drill, all-codex) landed partial-accepted at friction 3.

Schema gap → fix #8: land records must stamp per-stage worker.model.

dToken & active-time economics are unmeasuredCaveat

cost.tokens and cost.wall_s are 0/319 populated; active-time is 0s everywhere (no Codex activity logs to attribute). Wall time is the only usable time signal. Any "ceremony cost in tokens" claim is currently unsupportable.

eThe friction scale is defined two opposite waysBug

The ledger's plan_friction reads 5 = plan survived cleanly (20/29 scored 5, nearly all 0-rework + accepted). But harness-evaluation.md defines 1 = frictionless … 5 = blocked — the opposite direction. Same word, inverted scales. Any dashboard mixing them ranks backwards.

Cheap fix #17; flagged loudly because it silently corrupts downstream aggregates.

Top-5 — concrete diffs / specs

The five highest-leverage updates, with implementable specs. All target the agency repo and are Batu-gated — this card writes the spec, not the code.

1

Per-project merge gate [twf]

F2F8idea 1every TS landing (22 cards)

twf.py:_run_verification_gate (line 1240) hardcodes uv run python -m pytest -q + uv run ruff check. Every TS-repo landing fails it; the conductor runs the npm gate by hand. Stale node_modules after dep-adding merges caused false-fails (F8).

Spec. Read the gate from agents/config.json, run an optional pre-step first, fall back to the Python default when unconfigured.
// agents/config.json "twf_gate": { "pre": ["npm install --no-audit --no-fund"], // F8 "cmds": ["npm run typecheck", "npm run lint", "npm run test:unit", "npm run audit"] }
Load config in _run_verification_gate; if twf_gate present, run pre + cmds (via shlex.split) in order, else the current pytest/ruff pair. Add a test that a configured gate runs in order and a missing config falls back.
2

Honor card classification + collapse non-applicable gates [twf][card-tmpl][skill]

F1F3F16idea 2/3

Pickup routing is nondeterministic (F3); one-pass cards are marched through 6–8 gates (§1b), some demanding artifacts that make no sense for the class (F16).

Spec (two parts). (1) Card carries its own classification + profile; twf pickup/run-card read the body when the flag is omitted instead of guessing:
Classification: direct-to-work # | needs-plan | needs-brainstorm Pipeline: short # full | short | analysis
(2) Profiles define the set of applicable columns; twf next auto-stamps "n/a for <profile>" and skips the rest:
PIPELINE_PROFILES = { "full": PIPELINE_SEQUENCE, "short": [todo, planned, worked, reviewed, merged], "analysis": [todo, worked, reviewed, merged], # THIS card }
The analysis profile is exactly what this review card wants. Subsumes idea 3 + half of idea 2; ship #7 as the manual escape hatch in the same PR.
3

Contract-ownership line + round-trip test for cross-card seams [card-tmpl][skill]

F5idea 9§1a rework tax

The sdk owned-mirror sink and services worker shipped incompatible wire shapes under one schema tag — each card green, mismatch cost a full extra pass (F5). §1a shows the rework landing on exactly these seam cards. No gate caught it.

Spec. Card-writing checklist (twf-conduct + template): if a card produces/consumes a contract another card touches — (1) name the shared file in both bodies: Contract: packages/sdk/src/analytics/wire.ts (owner: this card); (2) declare exactly one owner, consumers import; (3) acceptance = a zero-adaptation round-trip test (producer emits → consumer parses, no massaging — the proven analytics-wire pattern). Optional lint: twf warns if two in-flight cards both claim owner: this card for the same path.
4

merge-card: auto-ledger + no-hand-merge guard + refusal hints [twf]

F7F7bF8idea 4/5/8

Conductor forgets twf ledger add under load (sparse outcomes, §1c); two premature "Landed" comments before the gate ran (F7); a hand-merge after refusal hit the truncated-branch trap (F7b).

Spec (three changes to run_merge_card). (1) Auto-append the land record after gate-pass — it already has card, stage (landed["stage"]), verify result; stamp worker.model from the stage's spawn record (closes idea 8 and the §1c gap → #8). (2) "Landed" comment only after gate-pass, from inside merge-card (idea 5) — assert unreachable before gate_fn returns 0. (3) Refusal text: print the exact resolving command, end with "re-run twf merge-card <id> — do NOT hand-merge (F7b)"; add --fix-dirty for lockfile-only auto-commit.
5

Shared-utils preflight in multi-subtree cards [card-tmpl][skill]

F4idea 10real defect

with-timeout was vendored 3× inside one package in one night — a real defect, not churn — because parallel cards each added a subtree without a named home for shared helpers.

Spec. Card-writing checklist: when adding parallel subtrees to one package, name shared utils + canonical path up front — Shared: with-timeout lives at packages/sdk/src/with-timeout.ts — import, do not vendor. First card to land owns it; later cards import. Optional guardrail: extend the duplication linter to flag identical exported function bodies across sibling subtrees.

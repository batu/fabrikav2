Where the time and tokens went

All 2,313 session files touching fabrikav2 — 1,910 Claude Code transcripts, 285 spawned Codex workers, and 118 rollouts of the interactive gpt-5.6-sol conductor — mined for time, tokens, and what the calls were actually doing.

Verdict: the spend is ~60% object-level and warranted, ~40% coordination — and the coordination is concentrated, not smeared. Build/verify work (game code, tests, device evidence) dominates hours. But the single largest token line-item is the conductor itself: one long-lived Codex session (plus its 115 sub-agent forks) that consumed 2.8B tokens across ~12k main-thread API requests, 78–84% of whose tool calls are board bookkeeping and output-harvesting glue. (Deep dive on that session.)

The second concentration is rework: 4 cards out of 162 consumed 28% of all pipeline events (and 84% of rework rounds). The review gates are earning their keep — the U1 adversarial audit caught four real false-passes — but nothing caps how many times a card can go around the loop.

Optimizations · ranked by leverage

  1. Put the conductor on a context diet. Reset the TUI session per work block (morning board pass, per-card arc) instead of resuming for days; keep board state in files, not conversation history. At 176k average input per request, every avoided turn-of-history saves real money and latency even at 98% cache hit.
    Target: −40–60% of the 2.8B conductor line; also the biggest latency win.
  2. Make twf emit digests so the conductor stops harvesting. 7,220 JS-glue calls exist because worker output arrives raw. A deterministic twf digest <card> (≤20-line JSON: verdict, diff stat, failing tests, evidence links) replaces most of them — plain code doing what premium context currently does.
    Removes most of the 42% glue-call share and shrinks what enters conductor context.
  3. Cap handoff prose. 2.80M output tokens — the fleet's #1 writing category — went into twf handoffs/comments. Enforce the existing Done/Verified-how/Remaining/Surprises schema with a hard line budget (say 12 lines + links); reject longer in the CLI.
    ≈1.5–2M output tokens/week back, and shorter handoffs also shrink conductor input.
  4. Add a rework circuit-breaker. After 3 rework rounds on one card, auto-park with evidence and a needs-human ping. Four cards burned 69 of 82 rework rounds; a cap converts runaway loops into one cheap human decision.
    Reclaims a large slice of the 28% of pipeline events spent on 4 cards.
  5. Gate re-reviews to the failed dimension. Review events ≈ build events today. Re-run only the dimension that failed (aesthetics, policy, seam) instead of full-stack re-review; keep the adversarial audit but trigger it by risk class (policy/money/publishing), not routine.
    DUAL U2 alone logged 28 review events; most re-ran ground already covered.
  6. Stop polling the board. 6,291 board-ceremony calls ≈ one every 27 active seconds. The fleet-bus / Portal doorbell already exists — subscribe for card-state changes and read one cached sitrep snapshot per wake instead.
    Thousands of calls and their tool-results removed from the priciest context.
  7. Label sessions on change, not on schedule. 1,704 haiku one-shots (74% of all session files) re-label the fleet. Hash the recent activity; skip the call when unchanged.
    Small token savings, big observability cleanup — session dirs stop drowning in label noise.
  8. Route mechanical stages to cheap models. evidence_captured / merged / compounded bookkeeping runs fine on Sonnet or Haiku; today Opus and gpt-5.5+ do nearly everything. This is the same direction as the existing TWF budget router — wire stage-class into it.
    Shifts a chunk of the 13.5M worker output tokens to models ~5–10× cheaper.
  9. Fence the conductor's hands. 592 direct worktree patches came from the most expensive seat. Anything beyond a one-line fix goes through a spawned worker whose context starts clean and dies after.
    Keeps repair diffs (and their file reads) out of the long-lived conductor history.

Appendix · method & caveats

How this was measured
  • Sources: ~/.claude/projects/*fabrikav2* + 71 card-worktree dirs (1,910 transcripts, 460MB); ~/.codex/sessions rollouts with a fabrikav2 cwd (285 files, 323MB) plus 118 conductor rollouts at the appletolye root that mention fabrikav2 (719MB); the agency ledger (567 fabrikav2 events) for card loops. Card→repo mapping came from the ledger.
  • Active time = sum of inter-event gaps ≤10 min. Tool exec time = tool_use→tool_result deltas, capped 30 min.
  • Conductor totals use fork-seed accounting: sub-agent forks replay parent history with re-stamped timestamps and inherit its token counter, so each fork's own spend = final counter − parent counter at fork time. Naive per-file summing reads 68.7B; the honest figure is 2.82B. (An earlier revision of this page said 3.45B — timestamp dedup still over-counted re-stamped replays; see the deep-dive appendix.)
  • Output-token attribution splits each assistant message evenly across the tools it invoked; "prose" = messages with no tool call. Command classification is regex-based on the command text.
Caveats
  • Classification is heuristic; compound shell commands take the highest-priority match. Code-mode JS snippets that both read the board and patch files count once (first match).
  • Conductor token totals may include usage mirrored from its ephemeral sub-agents (14,649 sub-agent activity events); fork-own spend is computed by counter deltas, so the 2.82B era figure is tight, not an upper bound.
  • Tool-exec hours (~15h Claude-side) exclude model generation latency, which dominates active time; time conclusions here lean on active-hours and call counts, not exec seconds.
  • No dollar figures: two providers, several tiers, and cache pricing differences would make a single number false precision. Volumes above are exact; multiply by your own rates.
  • 07-13 is a partial day; the pipeline was still running (GRAPES 2.5/8) when mined.
